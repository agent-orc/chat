Chat UI components for coding agents

An Angular library that renders an agent's event stream as a readable conversation — messages, tool activity, waits and decisions. Built as the frontend for coding-agent-runner; it works with any backend that emits the same events.

Overview

What's in the library

23+ event kinds · 7 entry points · 379 green specs · zero Angular imports in core · Apache-2.0

Event kinds

One append-only format tells the whole story — you already saw it above: message.user, a toolBurst of nine calls, a supervisor.wait while the agent went quiet, the orchestrator's retry decision. 23+ kinds in one union type; unknown kinds render as a safe fallback row.

Framework-free core

coding-agent-chat/core is plain TypeScript: the ConversationEvent contract plus projectConversation(). Backends, SSR code and tests can import it without Angular.

Entry points

Seven import paths, so you ship only what you render — the wire contract alone (core) is plain TypeScript with zero Angular in it. Every entry point has a live, touchable sample in the explorer below ↓

Host integration points

Four injection tokens with safe defaults — wire nothing and everything still renders. One concrete example: provide CHAT_MEDIA_LIGHTBOX and every screenshot in a conversation becomes click-to-enlarge — exactly what this page does ↑

Theme

One optional CSS file provides the look used on this page — dark by default, light by setting data-studio-theme="light" on any parent element.

Zoneless & OnPush

Signal-based components with strict templates, shipped as a partial-Ivy package. This page runs without Zone.js.

Tests

246 component specs run against the built package — the same output you install — plus 133 specs for the projection and markdown utilities.

License

Apache-2.0, the same license as the backend counterpart coding-agent-runner.

Host integration points

Each integration point has a default, so nothing is required to start: task keys render as plain text, images do not zoom, the history list is empty, deep history loads without a prompt.

CHAT_TASK_REFERENCE_PROVIDER markdown

Supplies task references for markdown auto-linking and handles navigation when one is clicked. Default: task keys render as plain text.

CHAT_MEDIA_LIGHTBOX shared

Owns the click-to-enlarge image overlay (modal stack, focus trap). Default: images do not zoom.

PROJECT_CHAT_DATA_SOURCE history

The scroll/search/stats/turn transport behind <cac-project-chat-list>. Default: an empty history.

CHAT_HISTORY_CONFIRM history

Confirmation prompt before loading an entire deep history. Default: auto-confirm.

Theming

The theme is one optional stylesheet in the package. Import it and set data-studio-theme="light" on a parent element for light mode — that is what the theme toggle in the demo does.
