Why this library

Built like a product, packaged like a library

23+ event kinds

One append-only wire grammar from message.user to workbench.debug: tool bursts, watchdog waits, orchestrator decisions, needs-input, artifacts, token metrics, task & run markers, trace links.

Zero-Angular core

@coding-agent/chat/core is pure TypeScript: the ConversationEvent contract plus projectConversation() — importable by backends, SSR and tests with zero renderer weight.

7 entry points

Pay only for what you render: core, markdown, conversation, composer, history, shared — and the root with provideCodingAgentChat().

Seams, not couplings

Task-reference linking, image lightbox, history transport and deep-history confirm are injection tokens with safe no-op defaults — the components work before you wire anything.

Drop-in studio theme

One optional CSS file ships the whole look: dark by default, light via data-studio-theme="light" on any parent. This page — including both demos — runs on it.

Zoneless & OnPush

Signal-first components, strict templates, partial-Ivy dist. The demos on this page run without Zone.js — change detection is driven entirely by signals.

379 green specs

246 component specs run against the built package (the same partial-Ivy output you install) plus 133 pure kernel specs on the projection and markdown utils.

Apache-2.0

Permissive license, no strings attached — same as its backend counterpart coding-agent-runner.

Entry points

Six secondary entry points keep the dependency graph honest — importing the wire contract never drags in the renderer.

Import Contents
@coding-agent/chat everything + provideCodingAgentChat()
@coding-agent/chat/core wire contract + projection + pure helpers (zero Angular)
@coding-agent/chat/markdown <cac-markdown> + markdown utils + task-reference seam
@coding-agent/chat/conversation <cac-conversation-view> + tool-burst chip + session card
@coding-agent/chat/composer <cac-chat> composer + role badge + workforce/phase helpers
@coding-agent/chat/history <cac-project-chat-list> virtualised history + <cac-chat-row> + minimap rail + phase summary strip
@coding-agent/chat/shared cacTooltip, cacStickToBottom, lightbox directive + tokens

Host seams — every one defaults to a safe no-op

CHAT_TASK_REFERENCE_PROVIDER markdown

Supplies task references for markdown auto-linking and handles navigation when one is clicked. Default: task keys render unlinked.

CHAT_MEDIA_LIGHTBOX shared

Owns the click-to-enlarge image overlay (modal stack, focus trap). Default: images do not zoom.

PROJECT_CHAT_DATA_SOURCE history

The scroll/search/stats/turn transport behind <cac-project-chat-list>. Default: an empty history.

CHAT_HISTORY_CONFIRM history

Guard prompt before loading an entire deep history. Default: auto-confirm.

Theming

The studio look ships as one optional stylesheet in the package. Import it, and flip a single attribute for light mode — exactly what the theme toggle on the live demo above does.
