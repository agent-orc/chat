(b) provideCodingAgentChat() ist in README und public-api-Kommentar versprochen, aber nicht implementiert. (c) README-Entrypoint-Tabelle vervollständigen (nur 2 von 6 gelistet), obsolete devDependency @types/dompurify entfernen.

Aufwand: klein · Ergebnis: Paket technisch publishfähig

Test-Harness + Komponenten-Specs angekündigt

In vier Commit-Bodies als „follows“ angekündigt: TestBed-Wiring für die Angular-Komponenten. Aktuell testen die 123 Specs nur den puren Kernel — conversation, composer, markdown-Komponente und shared-Directives haben null Abdeckung.

Aufwand: mittel · Ergebnis: Refactoring-Sicherheit vor der Adoption

ChatRow/History-Phase angekündigt

Im Phase-4c-Body explizit als eigene Phase angekündigt: chat-row und den Projekt-Chat-Verlauf (project-chat-rail, virtualisierte project-chat-list inkl. Load-Strategy) extrahieren. Dabei chat-row auf die Lib-RoleBadge umstellen, sonst entstehen zwei Role-Badge-Wahrheiten.

Aufwand: mittel · Ergebnis: Chat-UI-Fläche der Lib vollständig
4

Conversation Lab + npm-Publish 0.1.0

Die im README angekündigte Demo-/Playground-App als zweites Workspace-Projekt anlegen (Fixtures aus core wiederverwenden) — sie ist zugleich der Konsumenten-Smoke-Test gegen dist/. Danach Release-Workflow analog zum Runner: Version-Tag → GitHub Actions → npm (Trusted Publishing/OIDC gibt es auch bei npm), --access public für das scoped Package.

Aufwand: mittel · Ergebnis: installierbares @coding-agent/chat@0.1.0
5

Host-Adoption im Agent-Studio-Frontend Schlussstein

Analog zum Runner-Muster (Block 6, heute für NuGet abgeschlossen): App konsumiert @coding-agent/chat, die ~20 Duplikat-Dateien werden gelöscht, Import-Aliase leiten auf die Lib um. Host-seitig bleiben: SignalR-/Polling-Anbindung, Seam-Implementierungen (Task-Navigation, Lightbox-Viewer), Diff-Viewer, CLI-Console, Rich-Editor, Attachment-Upload. Erst dieser Schritt beendet die Doppelpflege.

Aufwand: groß · Ergebnis: eine Quelle der Wahrheit, Migration abgeschlossen
Empfohlene Reihenfolge: 1 → 2 → 3 → 4 → 5. Schritt 1 ist klein und entsperrt alles Weitere; 2 vor 5, damit die Adoption gegen getestete Komponenten läuft. Solange 5 aussteht: Chat-Änderungen im App-Frontend immer in beide Richtungen spiegeln (heiße Dateien: conversation-view/*, conversation-projection.ts, chat.component.ts, conversation-event.ts).
